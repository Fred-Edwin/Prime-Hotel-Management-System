**Deviations from the phase plan / data model:**
1. **`items.low_stock_threshold` is a new real schema column**, not envisioned in `04_PHASE_PLAN.md` Phase 7's scope or `01_DATA_MODEL.md`. Flagged explicitly and decided via human confirmation (option: "add a real schema column now" over a hardcoded constant or percentage-based rule) before building, per CLAUDE.md's "Building a real screen" recommend-improvements protocol and its mid-project schema-change rules. `01_DATA_MODEL.md` §2's `items` table SQL updated in this same phase with the change documented inline.
2. **Two chart forms were added to the admin dashboard** (a single-series net-profit-driver trend sparkline in the hero band; a two-bar Restaurant vs. Canteen comparison below), where `04_PHASE_PLAN.md` Phase 7 explicitly excluded "trend charts beyond period toggles." This was a deliberate, human-approved deviation (the user explicitly asked to prioritize visual polish over the original chart exclusion, mid-session) — not a silent one. Both charts follow the `dataviz` skill's method in full: form chosen by data's job (magnitude/comparison, not decoration), a validated 2-color palette derived from the existing brand hues (not the skill's generic 8-hue default, since Prime Hotel's brand only has 2 hues — gold + aubergine-family violet, both snapped into the palette validator's passing lightness/chroma band via `node scripts/validate_palette.js`), mandatory direct value labels on every mark (required because both chart colors carry a contrast WARN against their surface, which the dataviz skill's rules make non-dismissable without a labeling relief channel), no legend on the single-series line (title names it), status-color override (not categorical color) for a location at a net loss. Documented in `docs/design/02_PATTERNS_AND_CHECKLIST.md` §5's "Admin dashboard" entry as a real design-system extension, not an improvised one-off.
3. **A previously-undetected WCAG contrast failure in `MetricCard`'s `onDark`+`trend` combination was found and fixed.** Not a Phase 7 regression — the `onDark` variant existed since Phase 1 but had never actually been used with the `trend` prop until this phase's Net Profit card needed to show "Operating at a loss." Both `--color-status-success`/`--color-status-error` fail 4.5:1 text contrast against `--color-surface-dark` (2.7:1 and 3.1:1 respectively, confirmed via the dataviz skill's `contrast()` helper, not eyeballed). Fixed by adding `--color-status-success-on-dark`/`--color-status-error-on-dark` tints (same hue family, ~9:1 contrast) to `app/globals.css` and `MetricCard.module.css`, flagged to the human and confirmed via AskUserQuestion before implementing (not silently patched inline). Documented in `00_FOUNDATIONS.md` and `01_COMPONENTS.md` §4.10.
4. **`app/globals.css` was initially missed when adding the chart tokens** — the chart palette was first written only into `docs/design/00_FOUNDATIONS.md`'s documentation block, not the real stylesheet that components actually read from. Caught during visual verification (comparison bars rendered with `background-color: rgba(0,0,0,0)`, i.e. transparent, confirmed via `getComputedStyle` — not guessed from source), fixed by adding the same tokens to `app/globals.css`. A reminder that `00_FOUNDATIONS.md` is documentation of the token contract, not the contract's enforcement point — `app/globals.css` is the one that actually ships.

**Challenges faced:**
- **A real infinite API-call loop, found only by looking at network traffic, not by "it renders."** An early screenshot attempt showed the dashboard stuck on "Loading…" indefinitely. Assuming a data problem would have been the wrong diagnosis — `curl`ing the same route directly returned a correct `200` immediately. The actual cause, found by instrumenting a throwaway Playwright script with `page.on("requestfinished")`, was `next dev`'s Turbopack Fast Refresh/HMR looping (`/api/dashboard/summary` firing 2–3×/second, unbounded) against a dev server that had accumulated a lot of file-watch churn during this session's many edits. Confirmed genuinely dev-only (not a real `useEffect` bug in `DashboardClient`) by testing the exact same page against a clean production build (`pnpm build && pnpm start`): exactly 1 API call, no loop. This is why the visual verification in this file's screenshots was captured against a production server, not `pnpm dev` — a useful general lesson for any future "stuck loading" symptom in this repo: rule out dev-server HMR churn with a prod-build check before assuming the component logic is wrong.
- **Two `react-hooks/set-state-in-effect` lint errors** in both new client components — fixed by matching the exact pattern already established in `ExpensesClient.tsx`/`OrdersClient.tsx` (an inner `async function load()` called from the effect, not `setState` calls directly in the effect body) rather than inventing a new pattern.
- **The chart-palette-doesn't-exist-in-globals.css bug** (Deviations #4) — a genuine "wrote it in the doc, forgot the real file" mistake, caught only by looking at `getComputedStyle()` output on the actual rendered bar element (`rgba(0,0,0,0)`, transparent) rather than assuming the CSS variable resolved because the token name was correct. Consistent with CLAUDE.md's "verify, don't guess" rule for visual claims — reading the CSS Module and reasoning "this references `var(--color-chart-primary-light)`, so it should render gold" would have missed that the variable was undefined at the point of use.
- **Deriving a validated 2-color chart palette without the dataviz skill's default 8-hue set** — Prime Hotel's brand genuinely only has 2 hues. The skill's own palette validator (`scripts/validate_palette.js`) made this tractable: iterated candidate golds/violets from the aubergine/gold family until both a light-mode and dark-mode pairing passed lightness band + chroma floor + CVD separation, landing on the existing `--color-brand-accent-deep` gold (`#D19F48`/darkened to `#B8863A` for dark mode) plus a new validated violet (`#9B78BF`, passes both modes at the same value). Both pairings carry a contrast WARN (not a FAIL) against their surface, which per the skill's own rules is legal only paired with direct labels — hence every chart in this phase ships mandatory value labels, never color-only identification.
